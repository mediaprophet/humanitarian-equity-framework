Abstract

This document specifies the Humanitarian Equity Framework, a system for tracking, attributing, and compensating contributions to humanitarian Information and Communications Technology (ICT) projects. It enables transparent code inspectability while reserving rights until a sustainable compensation system is implemented, ensuring fair rewards aligned with human rights principles.

Status of This Document

This is an unofficial draft specification produced by Web Civics. It is not endorsed by any standards body and is intended for discussion and refinement. Feedback is welcome via the GitHub repository.

Introduction

The Humanitarian Equity Framework addresses the need for fair compensation in collaborative humanitarian ICT development. It distinguishes between open-source inspectability for transparency and the prevention of uncompensated labor, promoting sustainability and ethical practices.

This specification defines ontologies, processes, and licensing mechanisms using RDF, ODRL, and related vocabularies.

Ontologies

This framework defines several RDF ontologies for managing project rules, claims procedures, contributors, cost models, and licenses.

Project Rules Ontology

Defines criteria for "useful work" using RDFS classes like UsefulWork and properties like hasRelevanceToHumanitarianGoals.

@prefix : <http://mediaprophet.github.io/humanitarian-equity-framework/ontology/project-rules#> .
:UsefulWork rdf:type rdfs:Class .

Claims Procedure Ontology

Outlines processes for submitting and approving claims, with classes like ClaimProcess.

Contributors Ontology

Models contributors and their contributions using FOAF and PROV.

Cost Model Ontology

Defines obligation costs with classes like DirectCost and IndirectCost.

License Ontology

Represents the custom license in RDF, using ODRL for permissions and prohibitions.

:HumanitarianEquityLicense rdf:type :CustomReservedLicense ;
    dc:title "All Rights Reserved, Pending Compensatory Implementation" .

Licensing Mechanism

The framework uses a placeholder license that reserves rights until compensation criteria are met, then transitions to a permissive license.

Key sections include preamble, copyright reservations, limited rights, obligation costs, compensatory implementation, exemptions, metadata, no warranty, and governing law.

Implementation Considerations

Implementations SHOULD use blockchain for compensation and RDF for attribution. Contributions require approval via agreements, with pending status until approved.
