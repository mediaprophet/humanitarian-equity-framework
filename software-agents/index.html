This document provides a draft specification for a Web Civics Ontology, defining RDF-based schemas for rights, obligations, software agents, and acknowledgments in digital content creation. It emphasizes human-centric (biosphere) priorities, integrates infosphere concepts, and addresses implications of AI tools like LLMs, including fee-based access and role distinctions. This is an unofficial draft intended to foster humanitarian ICT development.

This is an Editor's Draft of an unofficial specification. It is not endorsed by any standards organization and is provided for discussion purposes only. Feedback is welcome via the associated GitHub repository.

Introduction

The Web Civics Ontology aims to provide a semantic framework for modeling rights and obligations in digital ecosystems, extending from human-centric views to broader infosphere considerations. It incorporates FOAF for agents, RDF for ontologies, and mechanisms for acknowledging AI contributions without implying ownership. Key factors include biosphere priorities (natural persons), work distinctions (e.g., 8-hour principles), and compensation via fees for AI access.

This ontology builds on existing standards like RDF, OWL, PROV-O, and FOAF, while introducing namespaces for infosphere, biosphere, AI agents, and acknowledgments.

Conformance to this ontology is non-normative but recommended for systems implementing semantic rights management. Implementations SHOULD use RDF/OWL tools for validation and reasoning.

Namespaces

The following namespaces are defined:

  • rights: http://example.org/rights/ - For intellectual property and status.
  • info: http://example.org/infosphere/ - For infosphere-specific rights and obligations.
  • bio: http://example.org/biosphere/ - For biosphere-centric entities and labor rights.
  • ai: http://example.org/ai/ - For software agents and resources.
  • ack: http://example.org/acknowledgment/ - For work acknowledgments and role distinctions.
  • Standard: rdf:, rdfs:, xsd:, foaf:, prov:, schema:, dct:

Core Classes and Properties

Human-Centric and Biosphere Classes

Prioritizing natural persons:

@prefix bio: <http://example.org/biosphere/> .

bio:BioEntity a rdfs:Class ;
  rdfs:label "Biosphere Entity" ;
  rdfs:comment "Living natural persons or entities as primary rights holders." .

bio:Human rdfs:subClassOf bio:BioEntity ;
  rdfs:label "Human" .

bio:LaborRight rdfs:subClassOf info:InfoRight ;
  rdfs:label "Labor Right" ;
  rdfs:comment "Rights for human work, ensuring fair pay." .

Infosphere Classes

Informational rights and obligations:

@prefix info: <http://example.org/infosphere/> .

info:InfoRight a rdfs:Class ;
  rdfs:label "Infosphere Right" .

info:InfoObligation a rdfs:Class ;
  rdfs:label "Infosphere Obligation" .

Software Agent Extensions (FOAF-Based)

Modeling AI agents and user requirements:

@prefix ai: <http://example.org/ai/> .
@prefix foaf: <http://xmlns.com/foaf/0.1/> .

ai:SoftwareAgent rdfs:subClassOf foaf:Agent ;
  rdfs:label "Software Agent" .

ai:LLM rdfs:subClassOf ai:SoftwareAgent ;
  rdfs:label "Large Language Model" .

ai:requiresPayment a rdf:Property ;
  rdfs:range xsd:decimal ;
  rdfs:comment "Payment for access, fulfilling compensation." .

Acknowledgment and Role Distinction

Distinguishing human authorship from AI facilitation:

@prefix ack: <http://example.org/acknowledgment/> .

ack:AcknowledgedWork a rdfs:Class ;
  rdfs:label "Acknowledged Work" .

ack:CreationRole a rdfs:Class ;
  rdfs:label "Creation Role" .

ack:AIGenerator rdfs:subClassOf ack:CreationRole ;
  rdfs:label "AI Generator" ;
  rdfs:comment "AI as tool; alternatives: Content Synthesizer, Generative Assistant." .

ack:createdWithAgent a rdf:Property ;
  rdfs:range ai:SoftwareAgent .

Fee fulfillment triggers rights:

ack:feeFulfilled a rdf:Property ;
  rdfs:range xsd:boolean .

Examples

Example instance for AI-assisted work:

project:UserGeneratedContent a ack:AcknowledgedWork ;
  ack:createdWithAgent project:Grok ;
  ack:distinguishesRole ack:HumanAuthor, ack:AIGenerator ;
  rights:ownershipRetainedBy project:HumanUser ;
  ack:feeFulfilled true .
